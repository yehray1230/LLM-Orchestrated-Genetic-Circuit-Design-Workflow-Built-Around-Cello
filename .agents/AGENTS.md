# AI Coding Assistant Guidelines

This document defines style guidelines, behavioral constraints, and safety limits for AI agents (such as Cursor, Claude, Gemini, Copilot, etc.) contributing to this repository.

## 1. Biological Claim Safety & Constraints

> [!IMPORTANT]
> **Safety Boundary**: You MUST NOT generate claims that any genetic circuit design generated by this computational workflow is experimentally validated, buildable, or guaranteed to function in vivo, unless external wet-lab verification data is explicitly provided by the user.

- Treat all ODE simulator and benchmark evaluator scores as **computational screening heuristics** rather than experimental success probabilities.
- Highlight biophysical uncertainties (e.g. RBS folding accessibility, metabolic burden, toxicity) explicitly in the agent outputs and user-facing reports.
- Refer to [limitations.md](docs/limitations.md) and [model_assumptions.md](docs/model_assumptions.md) before writing any biophysical claims.

---

## 2. Directory Layout & Import Conventions

- This repository uses the `src/` layout.
- All core Python files, schemas, utilities, agents, and templates must be placed inside the `src/` directory.
- **sys.path patches**:
  - Do NOT modify or remove the path overrides (`sys.path.insert(0, ...)`) at the top of:
    - `app.py`
    - `src/api/main.py`
    - `tests/conftest.py`
  - These patches allow imports from root level (e.g. `from schemas.state import DesignState`) to resolve successfully without prefixing them with `src.` inside the core codebase.

---

## 3. Developing and Testing

- **Testing**: Use `python -m pytest` to run the regression suite. Make sure `src/` is in your python path. You can set this up using:
  ```bash
  pip install -e .
  ```
- **Linter / Formatter**: We use `ruff` for code styling and import sorting. Always run `ruff check` before committing code changes.
- **Registries**: When modifying agent definitions or workflow configurations, run the registry verification script to make sure they compile:
  ```bash
  python src/scripts/build_registry.py
  ```

---

## 4. Keeping Documentation Current

- If you modify any core biological assumptions, limitations, or architecture models, make sure to rerun the documentation generator to rebuild `docs/llms-full.txt`:
  ```bash
  python src/scripts/generate_llms_txt.py
  ```
